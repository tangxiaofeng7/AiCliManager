# Security Policy

感谢你帮助 AiCliManager 提升安全性。

## 支持范围

当前安全支持范围以最新公开发布版本以及默认分支上的修复为主。

如果问题影响已公开发布版本，请在报告中明确受影响版本。

## 如何报告安全问题

请不要通过公开 Issue 披露漏洞细节。

建议通过以下方式私下联系维护者：

- GitHub Security Advisories / Private vulnerability reporting
- 仓库维护者公开的联系渠道

如果仓库后续补充专用安全邮箱，请优先使用该渠道。

## 报告中请尽量包含

- 漏洞类型与影响范围
- 受影响版本
- 复现步骤或最小复现条件
- 可能的利用前提
- 日志、截图或相关配置片段（请先移除密钥、令牌、密码等敏感信息）
- 你对修复思路的建议（可选）

## 处理原则

维护者会尽量：

- 确认问题是否可复现
- 评估影响范围与修复优先级
- 在修复发布前避免公开敏感细节
- 在修复完成后通过 Release 或提交记录说明影响与修复范围

## 敏感信息提醒

请不要在公开渠道粘贴以下内容：

- API Key
- 代理账号密码
- 本地配置文件中的明文敏感字段
- 可直接利用的攻击细节或 PoC
